# KG Open-Data Dashboard

Публичный статический сайт: ключевые индикаторы Кыргызстана с одного взгляда,
только из официальных открытых источников. Trust-first: нейтральный тон, без
мнений и прогнозов; у каждой цифры — ссылка на источник и дата данных.
Открытый репозиторий. Некоммерческий (данные НСК — CC BY-NC-SA, атрибуция
обязательна везде).

## Stack (зафиксирован)

- **Frontend:** Astro (SSG) + ECharts + Tailwind; MapLibre + GeoJSON регионов КР (фаза 2)
- **ETL:** Python 3.12 в `scripts/`, вывод JSON в `data/`
- **CI/CD:** GitHub Actions cron для обновления данных; деплой на Cloudflare Pages
- **i18n:** ru (основной) + ky (заглушки ключей допустимы, помечать TODO)
- **Требования:** mobile-first, первичная загрузка < 200 KB, тёмная тема, a11y

## Структура репозитория

```
scripts/parse_<metric>.py   # один парсер = один файл
data/<metric>.json          # выход парсеров (коммитится, "last good" для сайта)
tests/fixtures/<source>/    # один реальный сохранённый сэмпл на источник
tests/test_<metric>.py      # pytest на фикстурах
src/                        # Astro
src/i18n/{ru,ky}.json       # все строки UI
.github/workflows/          # daily.yml (02:00 UTC = 07:00 Бишкек), monthly.yml (НСК), ci.yml
```

## Схема данных (единая для всех метрик)

```json
{
  "metric": "string",
  "unit": "string",
  "source_name": "string",
  "source_url": "string",
  "fetched_at": "ISO-8601",
  "data_date": "YYYY-MM-DD",
  "series": [{ "date": "YYYY-MM-DD", "value": 0.0 }]
}
```

Расширение для годовых данных НСК (`parse_nsc.py`): `date` — строка года
(`"2024"`), плюс поле `regions` — все строки датасета (регионы/показатели)
с `name_ru/name_ky/name_en` и своей `series` для страниц деталей. `series`
верхнего уровня — ряд по Кыргызской Республике. Единицы измерения в JSON НСК
отсутствуют — берутся из официальных Excel той же категории (см. CATEGORIES
в parse_nsc.py); не выдумывать.

Правила пайплайна:
- Валидация: правдоподобные диапазоны значений + отсутствие разрывов дат.
- Ошибка fetch/parse → exit non-zero, CI падает громко.
- Сайт всегда показывает последний валидный JSON с бейджем «данные на <дата>».
- Никогда не выдумывать и не оценивать числа. Неясен URL/формат источника — остановиться и спросить.

## Метрики

Приоритет — данные НСК (stat.gov.kg) в доступной подаче. Проект — нейтральный
инструмент: без месседжа, оценок и интерпретаций; тексты карточек — только
определения показателей; дельты без цветового кодирования (только знак).

Готово:
1. **НСК, годовые** (`parse_nsc.py`, категории opendata): ИПЦ (127), зарплата (112),
   безработица (113), бедность (120), прожиточный минимум (119), население (316).
   Эндпоинт: `https://stat.gov.kg/ru/opendata/category/<id>/json`
2. **Курсы KGS** (USD/EUR/RUB/KZT), ежедневно — НБКР `nbkr.kg/XML/daily.xml`
   (windows-1251, запятая-разделитель); история — разовый бэкфилл `fin_stat.xls`

Дальше (по мере надобности): Токтогул (energo-es.kg), качество воздуха (OpenAQ),
другие категории opendata НСК.

## UI

- Главная — сетка карточек. Карточка: крупное число, Δ к прошлому периоду
  (без цветового кодирования — направление показывает знак), спарклайн
  (build-time SVG, без JS-библиотек на главной; переключатель 1г/5л только
  у ежедневных серий), одна строка-определение показателя, источник + дата данных.
- Страницы: `/` (дашборд), `/metric/<slug>` (полный график + таблица + скачать CSV),
  `/methodology` (источники, расписание, лицензия и атрибуция).
- Автогенерация OG-картинки на метрику с текущим значением.
- Все строки UI — через i18n-файлы, хардкода русского текста в компонентах нет.

## Рабочие правила

- Один парсер за раз: скачать реальный сэмпл → сохранить фикстуру → парсер → тесты → подключить карточку.
- Маленькие коммиты, conventional commits (`feat:`, `fix:`, `chore:`, `ci:`, `docs:`, `test:`).
- Числа не выдумывать. Источник неясен — спросить.
